Preserved paths · negations · steps

Compression is only useful when it is precise.

Wren is not trying to produce pretty summaries. It is trying to remove noise while keeping the tokens that silently change outcomes when they disappear.

Negations

Never, unless, and do not survive intact so the compressed result does not invert intent.

Values

Numbers, status codes, flags, file paths, and line references are treated as structural material, not optional decoration.

Branches

If a source block contains conditions, exceptions, or ordered steps, the compressed form keeps the control flow legible.

Tool Output

Read, grep, and build results behave differently from prompts, so Wren uses a separate output mode tuned for actionable traces.

Research Surface

Fine-tuned LoRA adapters, MLX-native inference, and separate prompt/output compression behavior. Small model, narrow task, pragmatic outcome.

Built like a narrow research tool, deployed like a utility.

The point is not to become another chat model. The point is to sit beside existing agents and make them less wasteful in the moments where verbosity crowds out useful context.

Training Prompt pairs and tool traces.

Wren is trained on compression pairs from real prompt blocks plus mined tool output. The examples optimize for constraint preservation, not style transfer.

Inference Local MLX on Apple Silicon.

The model loads once, stays hot, and compresses where the user already works. No hosted API, no extra privacy layer, no server bill.

Behavior Small enough to be specific.

The job is tightly defined: remove filler, keep structure, and refuse to win on compression ratio by throwing away the detail that actually matters.
